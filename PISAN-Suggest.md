## Project Summary

**UW Bothell Campus Pulse** is a real-time interactive map of UW Bothell that surfaces campus emergency alerts (parsed from the UW Alerts RSS feed by GPT-4o-mini) and personalized event recommendations (via a Surprise/SVD collaborative-filtering recommender with cold-start and popularity fallbacks). The stack is Next.js 14 + TypeScript + react-leaflet on the frontend, FastAPI + Python 3.11 + Supabase on the backend, deployed on Vercel + Render. The About page lists three team members (Kyle Hale, Nehemiah Soebroto, Andy Vu); the git history shows two distinct committers (`realkyle` with 9 commits and `Andy Vu` with 2), so Nehemiah's commit footprint is not visible.

1. **NLP alert summarizer** (`backend/nlp_summarizer.py`): GPT-4o-mini parses each raw RSS alert into a structured JSON object (`building_name`, `incident_type`, `severity`, `recommended_action`). Both are first-class application logic. **Estimated score: 13/15**. ## Suggested Improvements & New Features

### Code Quality & Technical

- Move the hardcoded fallback `https://css382-dyop.onrender.com` out of three files (`frontend/app/page.tsx` line 13, `frontend/components/LoginPanel.tsx` line 5, `frontend/components/RecommendationsPanel.tsx` line 5) into a single `frontend/lib/config.ts` that reads `NEXT_PUBLIC_API_URL` once. Vercel env vars are picked up at build time if set under "Production" scope before deploying; the issue in commit `76d1559` was likely a missing scope.
- Tighten CORS in `backend/main.py` line 21-26: restrict `allow_origins` to your Vercel production URL plus `localhost:3000`. Wildcard with `allow_credentials` would be unsafe; thankfully you do not use credentials, but it is still a habit worth fixing.
- Add `.DS_Store` and `**/.DS_Store` to `.gitignore`, then `git rm --cached .DS_Store backend/.DS_Store` to remove them from tracking.
- Add a `pytest` suite for `recommender.py` (cold-start, popularity fallback, CF threshold gating) and `building_coords.py` (resolution, partial-match fallback). These are pure functions and trivial to test.
- Replace inline `style={{...}}` with either CSS Modules or Tailwind. The slate-color palette is repeated 50+ times across components and is fragile. A `tokens.ts` constants file is the smallest possible step.
- Add a GitHub Actions workflow that runs `pytest` (backend) and `next build` (frontend) on every push to `main` — this catches deploy-breaking errors before Render/Vercel does.
- The `print(f"[poll_loop] error: {e}")` calls in `main.py` line 128 and across `database.py` should use the `logging` module so the Render logs are searchable by level.
- `nlp_summarizer.py` line 57-60 strips markdown fences manually. OpenAI's `response_format={"type": "json_object"}` parameter would eliminate this whole class of failure.

### UI / UX Improvements

- Add a public URL prominently to the README so graders and visitors do not have to guess. A "Live Demo" badge at the top would suffice.
- The footer legend on the main map (`frontend/app/page.tsx` lines 231-263) only explains severity colors. Add a legend chip explaining what each pin **shape** means once you add event pins, and what the "last updated" timestamp implies.
- The `/home` page embeds the live map in an `<iframe src="/" />` (line 102-107). This causes a double-load of the entire app and a nested header. Consider extracting `<CampusMap>` and rendering it directly, sized to fit the section.
- Mobile responsiveness is unverified. The map page uses a flex row with a 280px sidebar (`RecommendationsPanel.tsx` line 88) and the SiteNav is 200px (`SiteNav.tsx` line 21). On a phone, the map will be squeezed. Add a media query that collapses the For You panel into a drawer below ~768px.
- The Sign In modal accepts any string as a NetID with no validation. Add a regex check (e.g., `/^[a-z][a-z0-9]{2,7}$/`) and a clear error message.

### AI Integration Enhancements

- Move from a single-prompt classifier to **OpenAI function calling / structured outputs** so the JSON contract is enforced by the SDK rather than your regex-strip. This will let you drop the markdown-fence stripping in `nlp_summarizer.py` lines 57-60.
- **Add a "smart triage" layer**: when an alert is `severity=high`, have the model generate a 1-sentence push-notification headline and a 3-sentence safety brief. Two LLM calls instead of one, but the second only fires for high-severity events so cost is bounded.
- **RAG on past alerts**: store summarized alerts in Supabase (already wired) and, when a new alert arrives, retrieve the closest historical alert ("Last time UW1 had police activity, the all-clear came in 90 minutes"). Useful context for students and shows a more advanced AI technique.
- **Cluster the synthetic event catalog with embeddings** (text-embedding-3-small): replace the hand-built `MAJOR_TO_CATEGORIES` map with cosine similarity between the user's stated major and event descriptions. This generalizes to any major without code changes and demonstrates a second class of AI technique.
- **Recommender evaluation**: log offline metrics (precision@5, MAP) using a held-out split, even on synthetic interaction data. Shows you understand recommender quality, not just that it runs.
